Relatório Técnico — Teste de Desempenho de uma API de Checkout

1 Resumo Executivo

A capacidade máxima suportada difere drasticamente entre os dois perfis de carga. No cenário de I/O (POST /checkout/simple), por ser uma operação não bloqueante, a API sustenta pelo menos 300 usuários virtuais simultâneos (limite testado), com latência p95 de 483,80 ms (< 500 ms) e 0% de erro — o ponto de ruptura não foi atingido. No cenário de CPU (POST /checkout/crypto), o cálculo de hash bcrypt síncrono bloqueia a thread única do Node.js: a vazão satura em torno de 15 req/s e a capacidade útil é inferior a ~74 usuários virtuais. Conclui-se que o gargalo da aplicação é o processamento de CPU.
